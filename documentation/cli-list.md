# CLI Commands to Recreate BmoAtlas from Scratch

> These are the CLI commands needed to scaffold the entire BmoAtlas micro-frontend project.
> After running these commands, you would then manually create/edit the source files
> (components, services, styles, configs) as they are hand-written code, not generated by CLI.

---

## 1. Create the Angular Workspace

```bash
# Create a new Angular workspace with the shell application
ng new bmo-atlas --prefix=app --style=css --ssr=false --skip-tests

cd bmo-atlas
```

## 2. Restructure into Multi-App Layout

```bash
# Move the default app into apps/atlas/
mkdir -p apps/atlas
# (Move generated src/, public/, tsconfig.app.json, etc. into apps/atlas/)
```

## 3. Add Native Federation to the Shell

```bash
ng add @angular-architects/native-federation --project atlas --port 4200 --type dynamic-host
```

## 4. Generate the MFE Dashboard Application

```bash
# Generate a second application in the workspace
ng generate application mfe-dashboard --prefix=app --style=css --ssr=false --skip-tests

# Add Native Federation as a remote
ng add @angular-architects/native-federation --project mfe-dashboard --port 4201 --type remote
```

## 5. Generate the MFE Settings Application

```bash
# Generate a third application in the workspace
ng generate application mfe-settings --prefix=app --style=css --ssr=false --skip-tests

# Add Native Federation as a remote
ng add @angular-architects/native-federation --project mfe-settings --port 4202 --type remote
```

## 5b. Generate the MFE Stocks Application

```bash
# Generate a fourth application in the workspace
ng generate application mfe-stocks --prefix=app --style=css --ssr=false --skip-tests

# Add Native Federation as a remote
ng add @angular-architects/native-federation --project mfe-stocks --port 4203 --type remote
```

## 6. Install Runtime Dependencies

```bash
# es-module-shims — required by Native Federation for module-shim script loading
npm install es-module-shims
```

## 7. Install Dev Dependencies

```bash
# Concurrently — run all 3 dev servers in parallel
npm install --save-dev concurrently

# Wait-on — wait for MFE ports before starting the shell
npm install --save-dev wait-on

# Kill-port — free ports before starting dev servers
npm install --save-dev kill-port

# Rimraf — cross-platform clean command
npm install --save-dev rimraf

# Prettier — code formatting
npm install --save-dev prettier

# Vitest + Playwright — unit testing
npm install --save-dev vitest @vitest/browser-playwright @vitest/coverage-v8 playwright jsdom
```

## 8. Install Playwright Browsers (for Testing)

```bash
npx playwright install
```

## Files That Are Hand-Written (Not CLI-Generated)

After running the CLI commands above, the following files require manual creation/editing:

### Configuration
- `tsconfig.json` — add `@shared` and `@core` path aliases
- `angular.json` — add `polyfills: ["es-module-shims"]` to each app's esbuild options
- `package.json` — add `scripts` (dev, clean, kill-ports, start:*, build:*, test:*)
- `apps/atlas/federation.config.js` — shell federation config
- `apps/mfe-dashboard/federation.config.js` — expose `./routes`
- `apps/mfe-settings/federation.config.js` — expose `./routes`
- `apps/atlas/public/federation.manifest.json` — remote entry URLs
- `vitest.config.ts` — test runner config
- `.prettierrc` — formatting config
- `.editorconfig` — editor config

### App Configs (zoneless change detection)
- `apps/atlas/src/app/app.config.ts` — `provideZonelessChangeDetection()`
- `apps/mfe-dashboard/src/app/app.config.ts` — `provideZonelessChangeDetection()`
- `apps/mfe-settings/src/app/app.config.ts` — `provideZonelessChangeDetection()`

### Routing
- `apps/atlas/src/app/app.routes.ts` — shell routes with `loadRemoteModule()`
- `apps/mfe-dashboard/src/app/app.routes.ts` — dashboard child routes
- `apps/mfe-settings/src/app/app.routes.ts` — settings child routes

### Shared Library (`libs/shared/`)
- `libs/shared/src/index.ts` — barrel exports
- `libs/shared/src/styles/theme.css` — theme entry point
- `libs/shared/src/styles/_tokens.css` — design tokens
- `libs/shared/src/styles/_reset.css` — CSS reset
- `libs/shared/src/styles/_base.css` — base styles
- `libs/shared/src/styles/_layout.css` — layout utilities
- `libs/shared/src/styles/_components.css` — component styles
- `libs/shared/src/styles/_utilities.css` — utility classes
- `libs/shared/src/services/theme/theme.service.ts` — theme service
- `libs/shared/src/services/error/error.service.ts` — error service
- `libs/shared/src/components/error-toast/error-toast.ts` — error toast
- `libs/shared/src/components/atlas-loader/atlas-loader.ts` — loader
- `libs/shared/src/components/load-wrapper/load-wrapper.ts` — load wrapper (HttpData)
- `libs/shared/src/components/load-wrapper-client-data/load-wrapper-client-data.ts` — load wrapper (HttpClientData)
- `libs/shared/src/handlers/global-error.handler.ts` — global error handler
- `libs/shared/src/interceptors/error.interceptor.ts` — HTTP error interceptor
- `libs/shared/src/interceptors/mock-api.interceptor.ts` — mock API interceptor

### Core Library (`libs/core/`)
- `libs/core/src/index.ts` — barrel exports
- `libs/core/src/base/component/component-base.ts`
- `libs/core/src/base/domain/domain-base.ts`
- `libs/core/src/base/hub/hub.ts`
- `libs/core/src/base/service/service-base.ts`
- `libs/core/src/services/http-client-data/http-client-data.ts`
- `libs/core/src/services/http-data/http-data.ts`
- `libs/core/src/services/message-hub/message-hub.ts`
- `libs/core/src/services/state-hub/state-hub.ts`

### All Component Templates & Styles
- Every `.ts` component file (inline templates and styles)
- `apps/atlas/src/index.html` — FOUC prevention script
- `apps/mfe-dashboard/src/index.html` — FOUC prevention script
- `apps/mfe-settings/src/index.html` — FOUC prevention script
